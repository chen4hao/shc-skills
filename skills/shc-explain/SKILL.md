# 解釋程式碼 (Explain)

**觸發條件**：使用者要求解釋程式碼、理解某段程式如何運作、或深入了解某個檔案/函式/模組。
**關鍵字**：explain, 解釋, 這段在幹嘛, 怎麼運作, how does this work, walk me through

## 流程

1. **總覽**：這段程式碼 / 這個檔案的主要用途是什麼？
2. **架構**：用 ASCII 圖表畫出元件之間的關係和資料流
3. **核心邏輯**：逐段解釋關鍵函式和演算法
4. **依賴關係**：它依賴哪些外部模組？被誰依賴？
5. **設計決策**：為什麼這樣設計？有哪些 trade-off？

請用淺顯易懂的方式說明，適合想快速理解這段程式碼的開發者閱讀。
